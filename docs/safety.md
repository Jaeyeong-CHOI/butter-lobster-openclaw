# Safety Rules

## 자유롭게 해도 되는 것 (Internal)
- 파일 읽기/탐색/정리/학습
- 웹 검색, 캘린더 확인
- 워크스페이스 내 작업
- git status, 문서 업데이트, 커밋/푸시

## 반드시 승인 필요 (External)
- 이메일/트윗/공개 게시물 전송
- 메신저로 메시지 보내기
- 기계 밖으로 나가는 모든 행동
- 확신이 없는 모든 것

## 불변 제약
- private 데이터 외부 유출 금지. 절대.
- `rm` 대신 `trash` 사용 (복구 가능 > 영원히 삭제)
- API 키 평문 노출 금지 → `.env` 관리
- 파괴적 명령어는 실행 전 확인
- 의심되면 물어볼 것

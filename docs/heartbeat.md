# Heartbeat & Cron Guide

## Heartbeat vs Cron 사용 기준

| 상황 | Heartbeat | Cron |
|---|---|---|
| 여러 체크 배치 | ✅ | |
| 대화 맥락 필요 | ✅ | |
| 타이밍 유연 (~30분) | ✅ | |
| 정확한 시각 필요 | | ✅ |
| 메인 세션과 격리 | | ✅ |
| 다른 모델/thinking 레벨 | | ✅ |
| 원샷 리마인더 | | ✅ |
| 채널 직접 전달 | | ✅ |

**팁:** 비슷한 주기적 체크는 HEARTBEAT.md에 배치. Cron은 정확한 스케줄/독립 작업용.

## Heartbeat 체크 항목 (하루 2~4회 로테이션)
- Gmail 긴급 메일
- Calendar 24~48h 일정
- 소셜 멘션
- 날씨 (외출 관련)

## 상태 추적
`memory/heartbeat-state.json`에 마지막 체크 시각 기록

## 알림 기준
- 중요 메일 도착
- 일정 2시간 이내 임박
- 뭔가 흥미로운 것 발견
- 8시간 이상 침묵

## 침묵 기준 (HEARTBEAT_OK)
- 심야 (23:00~08:00) — 긴급 아닌 한
- 사용자가 바쁜 상태
- 마지막 체크 후 변화 없음
- 30분 이내 재체크

## Proactive 작업 (승인 불필요)
- 메모리 파일 정리
- 프로젝트 상태 확인 (git status 등)
- 문서 업데이트
- 자체 커밋/푸시
- MEMORY.md 리뷰 및 갱신

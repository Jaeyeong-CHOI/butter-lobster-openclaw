# HEARTBEAT.md

## 로테이션 체크 (매번 가장 오래된 1개만 실행)

`memory/heartbeat-state.json`의 lastChecks 타임스탬프를 읽고,
가장 오래된 체크 1개만 실행. 전부 한꺼번에 돌리지 말 것.

---

### 1) 메모리 유지보수 (주기: 2시간)

1. 오늘 daily log (`memory/YYYY-MM-DD.md`) 없으면 생성
2. 이 세션에서 중요한 결정/작업이 있었으면 daily log에 기록
3. 3일 이상 된 daily log 중 MEMORY.md에 반영 안 된 중요 사항 있으면 갱신
4. 아무것도 없으면 스킵

---

### 2) Git 상태 (주기: 2시간)

1. 워크스페이스 루트 `git status` 확인
2. 커밋 안 된 중요 변경사항 있으면 자동 커밋 (메시지 자동 생성)
3. origin에 푸시 안 된 커밋 있으면 푸시
4. 알림은 불필요 — 그냥 조용히 처리

---

### 3) 플랜 진행 추적 (주기: 6시간)

1. `memory/plans/*.md` 읽기
2. 완료 표시 안 된 체크박스(`[ ]`) 중 실행 가능한 것 있는지 확인
3. 있으면 daily log에 "다음 할 일" 메모
4. 플랜 상태가 오래 멈춰있으면 (3일+) daily log에 기록

---

### 4) cron 상태 (주기: 6시간)

1. 활성 cron 잡 목록 확인
2. 최근 실패한 잡 있으면 알림
3. 정상이면 스킵

---

### 5) 패턴 추출 — 자동 스킬 학습 (주기: 12시간)

최근 daily log 3일분을 읽고, 반복되는 작업 패턴을 감지한다.

**감지 기준:**
- 같은 유형의 작업을 2회 이상 비슷한 방식으로 수행한 경우
- 매번 같은 실수를 하고 같은 방식으로 수정한 경우
- 특정 도구/명령어 조합이 반복적으로 성공한 경우

**감지 시 행동:**
1. `docs/patterns.md`에 패턴 추가 (성공 패턴 또는 실패 패턴)
2. 형식:
   ```
   ### [패턴 이름] (날짜)
   - **상황**: 어떤 작업에서
   - **방법**: 무엇을 했더니
   - **결과**: 어떻게 됐다
   - **교훈**: 다음에는 이렇게
   ```
3. 이미 기록된 패턴과 중복이면 스킵
4. daily log에 "패턴 추출: [이름]" 메모

**감지 안 되면 스킵. 억지로 만들지 말 것.**

---

### 6) 메모리 압축 — daily log 정리 (주기: 24시간)

오래된 daily log를 정리하고 핵심만 보존한다.

**절차:**
1. 7일 이상 된 daily log (`memory/2026-*.md`) 확인
2. 각 파일에서 MEMORY.md에 반영 안 된 중요 사항 추출
3. 중요 사항 있으면 MEMORY.md에 추가
4. 해당 daily log를 `memory/archive/`로 이동
5. 변형 파일명 (`2026-03-24-topic.md` 형태) 발견 시:
   - 해당 날짜 정규 파일에 병합 후 archive 이동

**보존 기준:**
- 최근 7일 daily log는 항상 유지
- 7-30일: 핵심만 MEMORY.md에 반영 후 archive
- archive 파일은 삭제하지 않음 (memory_search로 검색 가능)

**정리할 것 없으면 스킵.**

---

## 규칙

- **활성 시간**: 08:00-23:00 KST. 밖에서는 HEARTBEAT_OK
- **알림 최소화**: 대부분 조용히 처리. 알림은 cron 실패 같은 진짜 문제만
- **토큰 절약**: 로테이션으로 매번 1개만. 무거운 외부 API 호출 지양
- 상태는 `memory/heartbeat-state.json`에 저장

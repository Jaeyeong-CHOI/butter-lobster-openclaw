# Report format rules (canonical)

Apply these rules by default when drafting or editing technical reports.

## Terminology and naming

- Use one canonical project name throughout the document.
- Keep naming consistent across title, section headers, and tables.

## Author/order convention

- Preserve a fixed author order when one is provided by the user.
- If no author order is given, use neutral placeholders (`Author A`, `Author B`, ...).

## Metric convention

- Keep only core metrics requested for the report.
- Remove legacy metrics unless explicitly requested.

## Section policy

- Exclude legacy/obsolete experiment sections unless explicitly requested.
- Exclude completion-status tables unless explicitly requested.
- Insert a page break before the "Current direction" section.

## Table quality policy

- Prioritize table alignment and line-break readability.
- Fix cramped or wrapped cells in milestone/meeting tables before final export.

## Change handling

- If user gives conflicting instructions, prioritize user instruction and record the override.

# Report → PPT mapping

Use this mapping to convert a LaTeX report into a concise presentation.

## Recommended 8-slide baseline

1. Title + one-line objective
2. Problem / motivation
3. Method overview
4. Core mechanism or equation
5. Results summary
6. Practical impact
7. Limitations and follow-up work
8. Takeaways + next steps

## Design constraints

- 16:9 canvas
- Single visual language from report palette
- Cover and body must be visually distinct (hero cover, structured body)
- Clear title hierarchy and breathing room
- One message per slide
- Prefer bullets/cards over dense paragraphs
- Reuse report-like elements where possible: numbered section marker, blue underline, key/advisor/insight box tone, table header color
- Fixed metadata anchors when requested:
  - top-right date
  - bottom-left author
  - bottom-right presentation title

## Typography guidance

- Slide title: 34–44 pt
- Section subtitle: 22–28 pt
- Body bullets: 20–26 pt (max 3–4 bullets/slide)
- Footer/meta: 11–13 pt
- Use cloud-safe fonts for shared viewing (e.g., Arial/Calibri/Noto Sans KR family)

## Conversion rules

- Merge adjacent low-priority sections
- Keep only core metrics
- Move long technical derivations to appendix slides if needed
- If using slides-grab templates, pick only needed templates from `slides-grab-template-catalog.md` instead of forcing all templates

## Visual policy

- Use simple cards/containers for grouping
- Keep high contrast text
- Maintain consistent left alignment unless intentional emphasis

04

주제 1 — RAG가 뭐야?

쉽게 말하면

  • RAG는 AI가 바로 대답하지 않고, 먼저 자료를 찾아보고 대답하는 방식
  • 사람이 모르는 문제를 받으면 검색부터 하는 것과 비슷함
  • 그래서 검색과 답변 생성이 같이 중요함

왜 필요할까?

  • LLM은 말을 잘하지만 가끔 추측해서 틀릴 수 있음
  • RAG는 문서나 자료를 찾아보고 답해서 더 정확해질 수 있음
  • 특히 학교 규정, 회사 문서, 논문처럼 근거가 필요한 정보에 유용함
05

LLM이란?

아주 쉽게 말하면

  • LLM은 아주 많은 글을 읽고 다음 말을 예측하는 AI
  • 그래서 사람처럼 자연스럽게 문장을 이어 말할 수 있음
  • ChatGPT 같은 대화형 AI도 이런 종류에 속함

하지만 한계도 있음

  • 모르는 것도 아는 척하면서 답할 수 있음
  • 최신 문서나 내 자료는 바로 모를 수 있음
  • 그래서 필요할 때는 밖에서 자료를 찾아와야 함

비유: LLM은 말을 아주 잘하는 똑똑한 친구인데, 시험 범위 밖 질문을 받으면 자신감 있게 틀릴 수도 있는 친구라고 생각하면 이해하기 쉽습니다.

06

벡터검색이란?

키워드 검색과 차이

  • 키워드 검색은 같은 단어가 들어가야 잘 찾음
  • 벡터검색은 문장을 숫자로 바꿔 뜻이 비슷한 것을 찾음
  • 단어가 조금 달라도 의미가 비슷하면 찾을 수 있음

예를 들면

  • 검색어: 시험 일정
  • 문서 제목: 중간고사 날짜 안내
  • 단어는 다르지만 뜻이 비슷해서 벡터검색이 잘 찾을 수 있음

한 줄 정리: 벡터검색은 “같은 단어 찾기”보다 “비슷한 뜻 찾기”에 더 가깝습니다.

07

하이브리드 검색이란?

왜 같이 쓸까?

  • 키워드 검색은 정확한 단어를 찾는 데 강함
  • 벡터검색은 뜻이 비슷한 문장을 찾는 데 강함
  • 하이브리드 검색은 두 방법을 함께 사용하는 방식

쉽게 비유하면

  • 한 명은 정확한 단어를 잘 찾는 조교
  • 한 명은 비슷한 뜻을 잘 찾는 조교
  • 둘이 같이 찾으면 더 안정적으로 좋은 자료를 찾을 수 있음

예시: 검색어가 ‘시험 일정’인데 문서 제목이 ‘중간고사 날짜 안내’일 때, 키워드 검색과 벡터검색을 같이 쓰면 더 잘 찾을 가능성이 큽니다.

08

검색 방식 비교

세 방식은 서로 경쟁 관계라기보다, 장단점이 다른 도구라고 보면 됩니다

방식 잘하는 것 약한 점 예시
키워드 검색 정확한 단어, 고유명사, 코드, 숫자 찾기 단어 표현이 달라지면 놓칠 수 있음 "도서관 운영시간"
벡터검색 비슷한 뜻, 자연스러운 문장 질문 찾기 정확한 용어·약어·숫자에는 약할 수 있음 "시험 일정" → "중간고사 날짜 안내"
하이브리드 검색 정확한 단어 + 비슷한 뜻 둘 다 챙김 설정이 조금 더 복잡함 학교 공지처럼 표현이 제각각일 때 유리
09

왜 하이브리드 검색이 강할까?

질문 예시

질문: "시험 일정 알려줘"

  • 키워드 검색: "시험 일정표" 같은 정확한 단어를 잘 찾음
  • 벡터검색: "중간고사 날짜 안내"처럼 뜻이 비슷한 문서를 잘 찾음
  • 하이브리드: 둘 다 참고해서 더 관련 높은 문서를 고름

그래서 좋은 점

  • 정확한 공지 제목을 놓치지 않을 수 있음
  • 표현이 달라도 관련 문서를 찾을 수 있음
  • 실제 학교 자료처럼 표현이 섞여 있을 때 더 안정적임

RAG는 어떻게 동작할까?

1

질문 받기

사용자가 질문을 입력함

2

관련 문서 찾기

벡터검색이나 키워드검색으로 자료 탐색

3

찾은 문서 읽기

질문과 관련된 부분만 가져옴

4

문서 기반 답변

찾은 내용을 바탕으로 답함

11

RAG 예시

같은 질문이라도, 자료를 찾느냐 안 찾느냐에 따라 다르게 보일 수 있습니다

그냥 LLM만 쓸 때

질문: “학교 도서관은 방과 후 몇 시까지 운영돼?”

  • 학교 자료를 직접 찾지 않음
  • “보통 6시쯤일 거예요”처럼 추측할 수 있음
  • 틀려도 왜 틀렸는지 알기 어려움

RAG를 쓸 때

질문: “학교 도서관은 방과 후 몇 시까지 운영돼?”

  • 학교 안내문에서 운영시간 부분을 찾음
  • 예: “평일 18:30까지 운영”
  • 답변에 근거를 붙일 수 있음

학생 발표에서 RAG를 볼 때

꼭 들어야 할 것

  • 키워드 검색, 벡터검색, 하이브리드 검색 중 무엇을 비교하는가
  • 어떤 답을 정확한 답이라고 볼 것인가
  • 답뿐 아니라 근거 문서도 잘 찾는지 볼 것인가

질문 예시

  • 벡터검색을 쓰면 뭐가 더 좋아지나요?
  • 정확도는 어떤 방식으로 비교하나요?
  • 문서를 잘못 찾으면 답도 같이 틀어지지 않나요?

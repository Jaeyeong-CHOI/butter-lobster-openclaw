# Contributing

Thanks for contributing to **butter-lobster-openclaw**.

## Scope

This repo is a public starter for OpenClaw workspace structure, prompts, and reusable skills.

## Before opening a PR

1. Keep changes generic (no personal/project-sensitive data).
2. Avoid committing secrets/tokens/credentials.
3. Keep documentation concise and practical.
4. Prefer small, focused PRs.

## PR checklist

- [ ] No private identifiers, emails, or host/IP secrets
- [ ] README/docs updated when behavior changes
- [ ] File paths and commands are valid
- [ ] Changes are understandable and minimal

## Commit style

Use clear prefixes, for example:

- `docs:` documentation updates
- `feat:` new reusable starter capability
- `fix:` bug fix
- `chore:` maintenance/refactor

## Reporting issues

Use GitHub Issues with clear reproduction steps and expected behavior.

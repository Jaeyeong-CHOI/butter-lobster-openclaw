# Security Policy

## Reporting a vulnerability

If you discover a security issue, please open a **private security advisory** in GitHub (preferred).

If private advisory is not available, open an issue with minimal details and ask maintainers to follow up privately.

## What to include

- Affected file/path
- Impact summary
- Reproduction steps
- Suggested mitigation (if available)

## Response goals

- Acknowledge report within 72 hours
- Triage and classify severity
- Patch and publish fix guidance

## Notes

Please do not publish exploit details publicly before maintainers have a chance to patch.
